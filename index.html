API Specifications

Serendip provides a full-fledged API that enables developers a complete access to all Serendip data and operations.
A client_id must be obrained first in order to use the API.

There are two use cases for the use of Serendip API:

  • Public Plan - rate-limited access to public non-personalized data such as charts and DJ info.
    Developers can obtain a serendip-public client id by signing up here
  • Full Plan - to upgrade your plan to get a high-rate full access to our API, please email us at api@serendip.me

JavaScript SDK

Developers who wish to develop web-based app using Serendip API are encouraged to use our JavaScript SDK.
Serendip JavaScript SDK takes out the heavylifting of configuring and managing the API common aspects, such as authentication, API calls and UI aspects.
The JavaScript SDK is designed as an asynchronous service model upon the Deffered design pattern.
